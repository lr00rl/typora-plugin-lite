# Wider Plugin

为 Typora 编辑器提供类似飞书文档的三档书写宽度切换：

- `default`
- `wide`
- `full`

插件只保留快捷键和命令入口，不再在编辑区右下角显示常驻控件。

## 功能

- **三档宽度切换**：正文宽度可在 `default / wide / full` 之间切换
- **持久化模式**：关闭再打开 Typora 后，保持上一次选择
- **快捷键切换**：
  - `Mod+[`：向窄调整
  - `Mod+]`：向宽调整
- **命令入口**：
  - `Editor Width: Narrower`
  - `Editor Width: Wider`
  - `Editor Width: Default / Wide / Full`

## 宽度阶梯

三档宽度会根据当前 Typora 窗口计算，不绑定某一种显示器：

| 模式 | 14 寸典型窗口 | 27 寸大屏 | 适用内容 |
| --- | ---: | ---: | --- |
| `default` | 最高 860px，空间不足时自动缩小 | 860px | 长文阅读、普通写作 |
| `wide` | 约 1000–1090px | 最高 1180px | 技术文档、代码、普通表格 |
| `full` | 使用可用宽度，通常约 1100–1390px | 最高 1680px | 宽表格、架构图、日志和对照内容 |

窗口窄到无法容纳目标值时，较宽的模式会自动收敛到实际可用宽度，不制造横向溢出。

## 与 Sidenote 的兼容

如果同时启用了 `sidenote` 插件：

- `wider` 会自动检测右侧边注预留宽度
- `wide` / `full` 模式下正文放宽时，不会把边注挤回正文区域
- 在窄窗口下，仍然会退回更紧凑的正文宽度，避免布局溢出

## 交互说明

- `Mod+[`：向窄调整一档
- `Mod+]`：向宽调整一档
- 切换时会在底部显示简短提示，例如 `Editor width: Wide`

## 设计目标

这个插件不是单纯把 `#write` 拉满，而是优先保证：

1. 默认档保持适合长文的 860px 阅读宽度
2. 宽档在小屏和大屏之间响应式调整
3. 全宽档利用可用空间，但不会在大屏无限拉长
4. 与 `sidenote` 共存时只为边注增加外壳宽度，不挤压正文
5. 不额外占用或遮挡正文界面
