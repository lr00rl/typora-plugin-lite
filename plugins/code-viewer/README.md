# Code Viewer

让 Typora 能"打开"非 Markdown 的文本/代码文件（`.py` `.ts` `.go` `.sh` `.json` `.yaml`…），
以**只读**方式展示，而**绝不修改原文件**。

## 为什么需要

Typora 会把 `.py` 之类的文件当 Markdown 打开——于是 `# comment` 变成一级标题、`*x*` 变成斜体、
反引号被吃掉。这个插件把文件按原样、逐行、带行号地显示出来。

## 安全模型（重点）

**插件从不改动 Typora 的文档模型。** 它用 `platform.fs` 直接读磁盘上的原始内容，然后在编辑器
**上面盖一层只读面板**。因为编辑器自身的内容没有被动过，Typora 没有任何"未保存改动"可写——
自动保存、退出时回写、`Cmd+S` 都不可能覆盖你的原文件。

> 早期版本曾用 `reloadContent` 把包了围栏的副本塞进编辑器，结果文档变"脏"，退出 Typora 时被
> flush 回磁盘，污染了原文件。教训：**不碰文档**。

额外两层保险：

- 打开代码文件后调用 `File.markSaved()`，即使 Typora 自身的 markdown 往返把脏标记碰了一下也强制置净；
- 拦截 `File.saveUseNode` / `File.saveAndBackup`：对任何非 md 文件直接 no-op（键于实时文件名，
  与插件内部状态无关，渲染即使失败也不会写盘）。

## 行为

- 打开受支持的文本文件 → 只读面板（文件名 + 语言选择器 + `只读` 标记，逐行 + 行号 + 语法高亮）。
- **复制干净**：行号是 CSS 伪元素、空行高度来自生成内容——选区/剪贴板里只有真实源码，
  不会带出行号，也不会漏入零宽字符。
- **空白可视化**（可关）：vim `listchars` 风格——空格显示为 `·`、Tab 显示为 `»`。原始字符不动，复制不受影响。
- **缩进参考线**（可关）：按行的缩进深度在等宽网格上画出垂直对齐线。
- **语言选择器**：默认使用按文件名检测的语言；可手动切换以修正不匹配的高亮，选择按扩展名持久化（设置里的 `langOverrides`）。
- 打开 `.md` / `.markdown` → 正常可编辑编辑器，插件完全让路。
- 二进制文件、超过大小上限的文件 → 显示提示，不渲染。
- 超过行数上限 → 只显示前 N 行并提示。

## 设置

- **Enable**：总开关。
- **Show whitespace markers** / **Show indent guides**：可视化开关，即时生效。
- **Max file size (bytes)** / **Max lines**：预览上限。

## 与 Markdown 代码块共享的组件

空白符切分、缩进列计算、参考线背景生成与标记样式放在 `packages/core/src/codeblock/`，
fence-enhance 插件把它复用到 Markdown 文档的代码围栏上：围栏由 CodeMirror 实时驱动，
其 lineView 缓存了文本节点引用，注入 span 会破坏光标测量——所以围栏侧只做**样式级**复用
（缩进参考线背景 + `cm-tab` 的 » 覆盖），不注入任何文本节点。

## 已知限制 / 后续

- 高亮器是内置的轻量 tokenizer（注释 / 字符串 / 数字 / 关键字 / 内建 / 标记标签），不是完整 parser；
  生僻语言回退为通用模式（注释 + 字符串 + 数字），可选 Plain text 完全关闭。
- 只读；如需编辑请用别的编辑器，或后续加“另存为 .md”。
- 语言识别覆盖 60+ 扩展名；未知但是文本的扩展名会以通用模式显示（不会拒绝打开）。
